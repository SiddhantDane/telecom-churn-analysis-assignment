# Telecom Customer Churn Analysis

**Module:** Machine Learning and Visualisation
**Programme:** BSc Computer Science and Digitisation
**Assignment:** Machine Learning and Visualization for Customer Churn

A machine learning and data visualisation project analysing customer churn for a
telecommunications provider. The project covers data preprocessing, exploratory
data analysis, predictive modelling, customer segmentation, and interactive
dashboards built in Tableau.

---

## Dataset

The analysis uses a customer churn dataset of 440,832 records (after cleaning),
containing customer demographics, account details, service usage, and a binary
churn label. The fields include Age, Gender, Tenure, Usage Frequency, Support
Calls, Payment Delay, Subscription Type, Contract Length, Total Spend, Last
Interaction, and Churn.

## How to run the analysis

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Run the exploratory data analysis (Task 1)

```bash
python code/task1_eda.py
```

This cleans the dataset, runs exploratory analysis, and saves the cleaned data
(`churn_cleaned.csv`) along with figures 1–4.

### 3. Run the machine learning models (Task 2)

```bash
python code/task2_models.py
```

Run this **after** `task1_eda.py`, as it uses the cleaned dataset. It trains and
evaluates the models, performs customer segmentation, and saves figures 5–9 plus
the Tableau-ready dataset (`churn_for_tableau.csv`).

> **Note:** the dataset is large (~440k rows), so `task2_models.py` may take a
> couple of minutes to complete.

## Methodology

**Task 1 – Data preprocessing and exploration.** Missing values and duplicates
are handled, summary statistics and correlations are computed, and key patterns
are visualised using Matplotlib and Seaborn.

**Task 2 – Machine learning.** Two supervised classifiers (Logistic Regression
and Random Forest) are trained with an 80/20 train–test split and hyperparameter
tuning via grid search, then evaluated using accuracy, precision, recall, F1, and
ROC-AUC. K-Means clustering is used to segment customers into behavioural groups.

**Task 3 – Visualisation and storytelling.** Interactive dashboards and a Tableau
Story present churn across contract length, subscription type, and demographics.

**Task 4 – Prescriptive analytics.** Data-driven retention recommendations are
derived from the model findings and customer segments.
